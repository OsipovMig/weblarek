# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

# Проектная работа «Веб-ларек»

Магазин товаров для разработчиков (Web-larek). В приложении можно просматривать каталог товаров, добавлять их в корзину и оформлять заказ, указывая данные доставки и контакты.

**Стек технологий:** HTML, SCSS, TS, Webpack.

---

## Архитектура приложения

Приложение построено на основе паттерна **MVP (Model-View-Presenter)**:

- **Model**: классы для хранения и управления данными (бизнес-логика).
- **View**: классы для отображения интерфейса и взаимодействия с пользователем.
- **Presenter**: слой, который связывает модель и отображение через события.

---

## Данные и типы данных (Интерфейсы)

В приложении используются следующие основные интерфейсы:

### Товар (IProduct)

Описывает структуру данных товара, приходящую с сервера.

- `id: string` — уникальный идентификатор.
- `description: string` — описание товара.
- `image: string` — ссылка на изображение.
- `title: string` — название товара.
- `category: string` — категория.
- `price: number | null` — цена (может быть пустой).

### Покупатель (IBuyer)

Описывает данные, которые заполняет пользователь в формах заказа.

- `payment: TPayment` — способ оплаты.
- `email: string` — электронная почта.
- `phone: string` — телефон.
- `address: string` — адрес доставки.

### Ошибки валидации (FormErrors)

Объект, где ключом является поле формы, а значением — текст ошибки.
`type FormErrors = Partial<Record<keyof IBuyer, string>>;`

---

## Модели данных (Классы работы с данными)

### 1. Класс CatalogModel (Каталог товаров)

Отвечает за хранение списка всех доступных товаров и управление выбранным товаром.

- **Поля:**
  - `_items: IProduct[]` — массив всех товаров.
  - `_preview: IProduct | null` — товар, выбранный для предпросмотра.
- **Методы:**
  - `setItems(items: IProduct[])` — сохраняет полученный массив товаров.
  - `getItems()` — возвращает текущий список товаров.
  - `getItem(id: string)` — возвращает товар по его ID.
  - `setPreview(item: IProduct)` — сохраняет товар для модального окна.
  - `getPreview()` — возвращает выбранный для предпросмотра товар.

### 2. Класс BasketModel (Корзина)

Класс управляет товарами, которые пользователь выбрал для покупки.

- **Поля:**
  - `_items: IProduct[]` — список товаров в корзине.
- **Методы:**
  - `getItems()` — возвращает список товаров в корзине.
  - `add(item: IProduct)` — добавляет товар в массив корзины.
  - `remove(id: string)` — удаляет товар из корзины по ID.
  - `clear()` — полностью очищает корзину.
  - `getTotalPrice()` — возвращает общую стоимость товаров.
  - `getCount()` — возвращает количество товаров в корзине.
  - `isInBasket(id: string)` — проверяет, добавлен ли уже этот товар.

### 3. Класс BuyerModel (Данные покупателя)

Отвечает за хранение контактной информации и её валидацию.

- **Поля:**
  - `_payment: string`, `_address: string`, `_phone: string`, `_email: string` — данные покупателя.
  - `_errors: FormErrors` — объект с описанием ошибок валидации.
- **Методы:**
  - `setData(field, value)` — сохраняет значение в конкретное поле. Позволяет обновлять поля частично, не удаляя другие данные.
  - `getData()` — возвращает объект со всеми данными покупателя.
  - `clearData()` — сбрасывает все сохраненные данные.
  - `validate()` — проверяет данные. Возвращает объект с текстом ошибок для каждого пустого поля. Если объект пуст — данные валидны.

## Слой коммуникации (LarekApi)

Класс `LarekApi` обеспечивает взаимодействие приложения с сервером. Он расширяет базовый класс `Api` и предоставляет методы для получения каталога товаров и оформления заказа.

**Конструктор:**

- `cdn: string` — базовый путь к изображениям товаров.
- `baseUrl: string` — базовый URL сервера.
- `options?: RequestInit` — дополнительные параметры запроса (опционально).

**Методы:**

- `getProducts(): Promise<IProduct[]>` — получает список всех товаров с сервера.
- `orderProducts(order: IOrder): Promise<IOrderResult>` — отправляет данные заказа на сервер и возвращает результат оформления.

## Раздел: Описание событий приложения (Event Broker List)

Данный раздел фиксирует все типы событий, которые генерируются в слое Представления (View) при взаимодействии с пользователем и передаются через брокер событий (EventEmitter) в Презентер для изменения состояния моделей.

### 1. События Каталога и Карточек товаров

- `card:select` — Вызывается при клике на карточку товара в галерее. Передает объект с `id` товара. Сигнализирует Презентеру о необходимости открыть окно детального просмотра (превью).
- `card:toBasket` — Вызывается при клике на кнопку «В корзину» (или «Удалить из корзины») на карточке превью. Сообщает Презентеру о необходимости обновить состав Модели корзины.
- `card:delete` — Вызывается при нажатии на кнопку удаления (крестик) на строке товара в списке корзины. Передает `id` товара для его удаления из Модели корзины.

### 2. События Корзины и Окон-контейнеров

- `basket:open` — Вызывается при клике на иконку корзины в шапке сайта. Служит сигналом Презентеру отрендерить актуальный список товаров в корзине и открыть модальное окно.
- `order:open` — Вызывается при клике на кнопку «Оформить» внутри окна корзины. Переводит интерфейс на первый шаг оформления заказа.

### 3. События инпутов в Формах и Валидации

- `order.payment:change` — Вызывается при клике на кнопки выбора способа оплаты («Онлайн» или «При получении»). Передает выбранный метод (`card` или `cash`) в Модель покупателя.
- `order.address:change` — Вызывается при вводе символов в поле адреса доставки формы `order`. Передает значение инпута для динамической валидации.
- `contacts.email:change` — Вызывается при вводе символов в поле Email формы `contacts`. Передает значение в модель для валидации.
- `contacts.phone:change` — Вызывается при вводе символов в поле номера телефона формы `contacts`. Передает значение в модель для валидации.

### 4. События отправки форм (Submit) и системные хуки

- `order:submit` — Вызывается при нажатии кнопки «Далее» формы доставки. Сообщает Презентеру, что первая форма заполнена, и пора открыть форму контактов `ContactsForm`.
- `contacts:submit` — Вызывается при нажатии кнопки «Оплатить» формы контактов. Служит сигналом Презентеру для отправки собранного заказа на сервер через API.

## Раздел: Презентер (Presenter Layer)

Презентер является главным координирующим узлом приложения «Web-ларек». Он связывает слой бизнес-логики (Модели данных) со слоем отображения (Компоненты Представления).

### Архитектурный подход

Для реализации логики приложения выбран подход **Event-driven Presenter (Презентер на основе событий)**, размещенный непосредственно в основном скрипте приложения `src/main.ts`. Такой подход является оптимальным для одностраничного приложения (SPA), так как позволяет избежать избыточного усложнения кода, сохраняя слабую связанность компонентов (Low Coupling) за счет использования паттерна «Брокер событий» (`EventEmitter`).

Презентер не имеет прямого доступа к DOM-элементам и не хранит внутри себя состояние приложения. Его работа строится по следующему циклу:

1. Презентер подписывается на события интерфейса (View) и события моделей данных (Model).
2. При действии пользователя (клик, ввод) View генерирует событие.
3. Презентер перехватывает событие, обрабатывает его и вызывает соответствующие методы изменения данных в Моделях.
4. Модели данных изменяют своё состояние и эмитят ответное событие.
5. Презентер перехватывает событие модели, передает новые данные в методы `render()` компонентов Представления, и интерфейс обновляется.

### Схема прохождения основного пользовательского сценария

- **Загрузка каталога**: `LarekApi` возвращает данные -> Презентер передает их в `CatalogModel` -> Модель генерирует `items:changed` -> Презентер клонирует шаблоны, создает инстансы `CardCatalog` и передает их в `page.catalog`.
- **Добавление в корзину**: Клик на карточку витрины -> событие `card:select` -> Презентер открывает модалку с `CardPreview`. Клик «В корзину» -> Презентер вызывает `basketModel.add()`.
- **Оформление заказа**: Клик «Оформить» в корзине -> Презентер рендерит `OrderForm` в модалку. Поля ввода транслируют изменения в `BuyerModel`. При успешной валидации кнопка «Далее» активируется. На шаге «Контакты» (`ContactsForm`) происходит аналогичный процесс.
- **Покупка**: Нажатие кнопки «Оплатить» -> Презентер собирает единый объект заказа `IOrder`, отправляет его на сервер через `larekApi.orderProducts()`, рендерит компонент `Success`, очищает корзину и данные покупателя в моделях.

### Слой отображения (View)

Компоненты слоя отображения отвечают за отрисовку интерфейса, вывод данных из моделей на экран и трансляцию действий пользователя в виде событий. Все компоненты наследуются от базового класса `Component<T>`.

#### 7. Класс `LarekApi`

Специализированный сервис для работы с эндпоинтами проекта «Web-ларек». Реализует паттерн композиции — использует базовый `Api` как внутренний инструмент.

- **Назначение**: Получение сырых данных с сервера и отправка заказов.
- **Конструктор**:
  - `constructor(private _api: Api)` — Принимает экземпляр низкоуровневого класса `Api`.
- **Методы**:
  - `getProducts(): Promise<IProductResponse>` — Запрашивает полный сырой объект ответа со списком товаров с сервера без изменений.
  - `orderProducts(order: IOrder): Promise<IOrderResult>` — Отправляет сформированный объект заказа на сервер методом POST.

#### 8. Класс `Page`

Управляет выводом сетки товаров на главной странице.

- **Назначение**: Отображение каталога.
- **Конструктор**: `constructor(container: HTMLElement)` — Принимает элемент галереи страницы.
- **Поля класса**: `_gallery: HTMLElement` — Контейнер для карточек.
- **Сеттеры**: `set catalog(items: HTMLElement[])` — Отрисовывает массив DOM-элементов карточек.

#### 8.1. Класс `Header`

Управляет элементами шапки сайта.

- **Назначение**: Отображение счетчика количества товаров в корзине и вызов события открытия корзины.
- **Конструктор**: `constructor(container: HTMLElement, events: IEvents)` — Принимает контейнер шапки и брокер событий.
- **Поля класса**: `_counter: HTMLElement` — Счетчик, `_basketButton: HTMLButtonElement` — Кнопка корзины.
- **Сеттеры**: `set counter(value: number)` — Обновляет цифру количества товаров.

#### 9. Класс `Modal`

Универсальный компонент модального окна-контейнера. Является финальным классом (наследование запрещено).

- **Назначение**: Управление видимостью всплывающего окна и динамическая подстановка контента.
- **Конструктор**:
  - `constructor(container: HTMLElement, events: IEvents)` — Принимает контейнер модального окна `#modal-container`. Навешивает один раз слушатели для закрытия окна (клик по крестику и по оверлею).
- **Поля класса**:
  - `_closeButton: HTMLButtonElement` — Кнопка-крестик для закрытия.
  - `_content: HTMLElement` — Контейнер `.modal__content` для динамической вставки содержимого.
- **Сеттеры**:
  - `set content(value: HTMLElement)` — Полностью заменяет текущее содержимое `.modal__content` на переданный HTML-элемент.
- **Методы**:
  `_content` и генерирует событие `modal:close`.

#### 10. Класс `Card`

Базовый абстрактный класс для отображения карточки товара. Наследуется от `Component<ICardView>`.

- **Назначение**: Инкапсуляция общих для всех карточек элементов (заголовок, цена) и предоставление базовых сеттеров.
- **Конструктор**:
  - `constructor(container: HTMLElement)` — Принимает DOM-элемент карточки и кэширует общие узлы.
- **Поля класса**:
  - `_title: HTMLElement` — Элемент названия (заголовка) товара.
  - `_price: HTMLElement` — Элемент стоимости товара.
- **Сеттеры**:
  - `set title(value: string)` — Устанавливает текст названия товара.
  - `set price(value: number | null)` — Устанавливает цену товара. Если цена `null`, выводит текст `Бесценно`.

#### 11. Класс `CardCatalog`

Компонент карточки товара для отображения на главной витрине. Наследуется от `Card`.

- **Назначение**: Отображение товара в общей галерее и обработка клика по карточке.
- **Конструктор**:
  - `constructor(container: HTMLElement, actions?: { onClick: () => void })` — Принимает контейнер карточки и объект с коллбеком клика. Навешивает слушатель `click` на всю карточку.
- **Поля класса**:
  - `_category: HTMLElement` — Элемент категории товара.
  - `_image: HTMLImageElement` — Элемент изображения товара.
- **Сеттеры**:
  - `set category(value: string)` — Устанавливает текст категории и добавляет соответствующий CSS-модификатор цвета подложки на основе маппинга `categoryMap`.
  - `set image(value: string)` — Устанавливает ссылку на картинку с помощью встроенного базового метода `this.setImage`.

#### 12. Класс `CardPreview`

Компонент детального просмотра товара в модальном окне. Наследуется от `CardCatalog`.

- **Назначение**: Отображение подробного описания товара и управление кнопкой покупки/удаления.
- **Конструктор**:
  - `constructor(container: HTMLElement, actions?: { onClick: () => void })` — Принимает контейнер превью-карточки и объект с коллбеком для кнопки. Навешивает слушатель на кнопку покупки с остановкой всплытия (`e.stopPropagation()`).
- **Поля класса**:
  - `_text: HTMLElement` — Элемент длинного текстового описания товара.
  - `_button: HTMLButtonElement` — Кнопка добавления в корзину / удаления из нее.
- **Сеттеры**:
  - `set description(value: string)` — Устанавливает полный текст описания товара.
  - `set buttonText(value: string)` — Изменяет текст на кнопке («В корзину» / «Удалить из корзины»).
  - `set price(value: number | null)` — Переопределяет родительский сеттер цены: если цена товара равна `null` («Бесценно»), автоматически блокирует кнопку покупки атрибутом `disabled` и меняет её текст на «Не продается».

#### 13. Класс `CardBasket`

Компонент строки товара для списка внутри корзины. Наследуется от `Card`.

- **Назначение**: Отображение товара в корзине с указанием порядкового номера и кнопкой удаления.
- **Конструктор**:
  - `constructor(container: HTMLElement, actions?: { onDelete: () => void })` — Принимает элемент строки корзины и коллбек удаления. Навешивает слушатель клика на иконку-крестик.
- **Поля класса**:
  - `_index: HTMLElement` — Элемент порядкового номера строки в списке.
  - `_buttonDelete: HTMLButtonElement` — Кнопка-иконка для удаления товара из корзины.
- **Сеттеры**:
  - `set index(value: number)` — Устанавливает порядковый номер строки (1, 2, 3...).

#### 14. Класс `Form<T>`

Базовый абстрактный класс для всех форм в приложении. Наследуется от `Component<IFormState>`.

- **Назначение**: Управление состоянием полей ввода, валидацией, выводом ошибок и блокировкой отправки.
- **Конструктор**:
  - `constructor(protected container: HTMLFormElement & { name: keyof T | string }, protected events: IEvents)` — Принимает HTML-элемент формы и брокер событий. Навешивает слушатель `input` на всю форму для трансляции изменений полей, и слушатель `submit` с отменой стандартного поведения браузера (`e.preventDefault()`).
- **Поля класса**:
  - `_submit: HTMLButtonElement` — Кнопка отправки формы (`type="submit"`).
  - `_errors: HTMLElement` — Блок `.form__errors` для вывода текста ошибок валидации.
- **Сеттеры**:
  - `set valid(value: boolean)` — Управляет доступностью кнопки отправки (добавляет или удаляет атрибут `disabled`).
  - `set errors(value: string)` — Выводит текст ошибок валидации в элемент `_errors`.

#### 15. Класс `OrderForm`

Компонент первой формы оформления заказа (способ оплаты и адрес доставки). Наследуется от `Form<Partial<IBuyer>>`.

- **Назначение**: Сбор адреса доставки и управление кнопками выбора метода оплаты.
- **Конструктор**:
  - `constructor(container: HTMLFormElement, events: IEvents)` — Принимает элемент формы `name="order"`. Находит кнопки оплаты и навешивает на них слушатели клика.
- **Поля класса**:
  - `_cardButton: HTMLButtonElement` — Кнопка выбора оплаты онлайн («Онлайн»).
  - `_cashButton: HTMLButtonElement` — Кнопка выбора оплаты при получении («При получении»).
- **Методы**:
  - `selectPayment(method: 'card' | 'cash'): void` — Подсвечивает выбранную кнопку с помощью CSS-модификатора `button_alt-active` и деактивирует вторую, генерируя событие `order.payment:change`.

#### 16. Класс `ContactsForm`

Компонент второй формы оформления заказа (Email и Телефон). Наследуется от `Form<Partial<IBuyer>>`.

- **Назначение**: Сбор контактных данных пользователя перед отправкой заказа.
- **Конструктор**:
  - `constructor(container: HTMLFormElement, events: IEvents)` — Принимает элемент формы `name="contacts"`. Полностью опирается на базовую логику конструктора `Form`.

#### 17. Класс `Basket`

Компонент отображения корзины. Наследуется от `Component<IBasket>`.

- **Назначение**: Отображение списка выбранных товаров и их суммарной стоимости в модальном окне.
- **Конструктор**:
  - `constructor(container: HTMLElement, events: IEvents)` — Принимает элемент корзины. Навешивает слушатель клика на кнопку «Оформить».
- **Поля класса**:
  - `_list: HTMLElement` — Элемент списка `<ul>` (`.basket__list`) для вставки строк товаров.
  - `_total: HTMLElement` — Элемент `.basket__price` для вывода итоговой стоимости.
  - `_button: HTMLButtonElement` — Кнопка «Оформить».
- **Сеттеры**:
  - `set items(items: HTMLElement[])` — Принимает массив готовых DOM-строк `CardBasket`. Если массив пуст, блокирует кнопку «Оформить» и выводит текст "Корзина пуста".
  - `set total(value: number)` — Выводит общую сумму товаров в формате `X синапсов`.

#### 18. Класс `Success`

Компонент финального окна успешного оформления заказа. Наследуется от `Component<ISuccess>`.

- **Назначение**: Отображение подтверждения покупки и списанной суммы.
- **Конструктор**:
  - `constructor(container: HTMLElement, actions?: { onClose: () => void })` — Принимает элемент окна успеха и объект с коллбеком закрытия. Навешивает слушатель клика на кнопку «За новыми покупками!».
- **Поля класса**:
  - `_close: HTMLButtonElement` — Кнопка закрытия окна.
  - `_description: HTMLElement` — Элемент описания для вывода списанной суммы.
- **Сеттеры**:
  - `set total(value: number)` — Выводит текст в формате `Списано X синапсов`.
